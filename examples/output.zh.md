# 中文输出示例

> 输入：某 AI 写作产品 DAU 下降 20%，新增注册不变，请做归因分析（AI PM 业务二面）。

## 1. 核心判断
DAU 降 20% 但新增不变 → 问题大概率出在**老用户留存/活跃**，不是拉新。**第一步必须先排除"假异常"（埋点/口径/时间窗）**，再按"分子-分母 + 四象限"拆，最后给可验证的决策。

## 2. 结构化分析（按归因树）
**Step 0 排假异常**：DAU 统计口径、埋点是否改版、时间窗是否跨节假日/周末。成本最低、最常见，先排。

**Step 1 拆 DAU**：DAU = 新增活跃 + 老用户活跃。新增不变 → 锁定**老用户活跃下降**。

**Step 2 四象限定位老用户为何不来**：
| 维度 | 要看什么 | 看到什么 → 下什么结论 |
|---|---|---|
| 数据口径 | 埋点/版本发布记录 | 当天有发版 → 大概率体验回归 |
| 外部 | 竞品发版、节假日、渠道 | 竞品大版本 → 用户被分流 |
| 用户结构 | 新/老、渠道质量 | 近期新增是低质渠道 → 留存本就差 |
| 产品/系统 | 核心任务完成率、模型响应质量/延迟 | 生成质量↓或延迟↑ → 体验劝退 |

**Step 3 因果而非相关**：老用户活跃↓要能归因到具体动作——对比"用过新功能的老用户 vs 没用的"留存差异；排除大盘季节性。

## 3. 可执行建议
1. 先拉"老用户 7 日留存 + 核心任务（写作完成）完成率 + 模型 P95 延迟"三条曲线，定位拐点日期。
2. 拐点对齐发版记录 → 若当天发版，灰度回滚验证。
3. 若无发版 → 看是否"新鲜感消退/生成质量波动"，按模板/场景粒度看满意度。
4. 决策：体验回归 → 回滚止血；质量漂移 → 近 30 天数据重训 + 设 NDCG 周环比告警。

## 4. 面试防守点
- 会被追问：「**你怎么证明是这个动作导致的，不是大盘？**」→ 用"用过/没用过新功能"的留存对照 + 同比基线排除季节性。
- 会被追问：「**先看哪个？为什么？**」→ 先排埋点口径（成本最低的假异常），再拆分子分母锁定老用户，避免一上来就归因到产品。
- 别犯的错：直接说"我会看数据"——必须给出**排查顺序 + 每步的判断规则**。
