# Evaluation Rubric / 评估标准

Use this rubric to judge whether the skill output is good enough.

用以下标准判断输出是否合格。

| Dimension | Good output | Weak output |
|---|---|---|
| Grounding / 贴合上下文 | Uses the user's real JD/resume/project context | Generic, could apply to anyone |
| Structure / 结构 | Clear headings, workflow-driven reasoning | Random paragraphs, no flow |
| Evidence / 证据 | Distinguishes facts, assumptions, and gaps | Overstates or fabricates evidence |
| Actionability / 可执行性 | Provides next steps and practice points | Only gives abstract advice |
| Interview defensibility / 面试可防守 | Prepares for follow-up questions | Creates claims user cannot defend |
| Safety / 边界 | No impersonation, no fake facts | Pretends to be a real person or invents facts |

## Pass threshold / 通过标准

A result passes if it is specific, structured, grounded, and can be used immediately in the website workflow or interview preparation.

如果输出能直接用于网站 workflow 或面试准备，且具体、结构化、基于证据，即为通过。
